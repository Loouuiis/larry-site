Task center status groups

You mentioned "ongoing, completed, and not started." The backend has 6 statuses: backlog, not_started, in_progress, waiting, completed, blocked. How should these map to sections?

A

Three groups (your suggestion)

Not Started — backlog + not_started
Ongoing — in_progress + waiting + blocked
Completed — completed

Simple, scannable. Blocked tasks show a small "blocked" badge within the Ongoing group.

B

Four groups (split out blocked)

Not Started — backlog + not_started
In Progress — in_progress + waiting
Blocked — blocked
Completed — completed

Blocked tasks get their own section for visibility. Slightly more visual weight.

C

Three groups + "Blocked" as a filter badge

Same as A (three groups), but add a small filter pill at the top:
All Blocked (2) At Risk

Clean three groups + quick filter access for problem tasks.
