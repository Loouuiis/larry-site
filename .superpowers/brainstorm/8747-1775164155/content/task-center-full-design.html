Design details

Inline creation flow:

  • Click "+ New task" in any group → dashed-border row appears at bottom of that group
  • Title field is auto-focused. Type the task name.
  • Tab → cycles through priority dropdown, assignee dropdown, due date picker
  • Enter → saves with whatever fields are filled (title required, rest optional)
  • Esc → cancels and removes the row
  • Priority defaults to "Medium", assignee and due date default to empty

Groups:

  • Each group is collapsible (click header to toggle)
  • "Completed" starts collapsed by default
  • Empty groups still show with "(0)" count and the "+ New task" button
  • Group colours: Not Started = purple (#6c44f6), In Progress = amber (#f59e0b), Blocked = red (#ef4444), Completed = green (#22c55e)

Task rows:

  • Dot colour matches group colour
  • Priority badge: Low=green, Medium=blue, High=amber, Critical=red
  • Assignee name or "—" if unassigned
  • Due date as "MMM DD" or "—" if none
  • Uses existing CSS variables (--surface, --border, --text-1, --text-2, --cta)

API calls:

  • Create: POST /v1/tasks (title, projectId, priority, assigneeUserId, dueDate)
  • After creation: call refresh() to re-fetch project data and update overview counts
  • Status mapping: the inline row's group determines the initial status sent to the API
